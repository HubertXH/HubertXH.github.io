G1-日志说明

JVM中GC日志的参数配置

参数 说明
-Xloggc:/path/to/gc.log 日志存储路径
-XX:+UseGCLogFileRotation 是否开启GC日志自动切换(true /false)
-XX:NumberOfGCLogFiles= GC 日志保留的数量(num)
-XX:GCLogFileSize= GC日志切换的大小(100M)
-XX:+PrintGCDetails 打印详细GC日志
-XX:+PrintGCDateStamps 打印GC发生的真实日期及时间
-XX:+PrintGCApplicationStoppedTime 打印在垃圾回收期间应用停顿的时长
-XX:+PrintGCApplicationConcurrentTime 打印GC时程序的运行时间
-XX:-PrintCommandLineFlags 在日志中打印所有命令标识

LOG示例：

  • ①
    • 2021-06-27T16:02:10.150+0800:GC发生的实际时间
    • 2203.486:距离JVM启动到现在的时间毫秒值
    • (G1 Evacuation Pause) (young)：GC类型 young GC
    • 0.0041826 secs:此次GC耗费时间
  • ②
    • Parallel Time: 2.8 ms:此次GC 执行期间Stop The Word所花费的时间，从垃圾收集开始直到最后一个线程结束为止 2.8毫秒
    • GC Workers: 10: GC工作线程数，可以使用参数：XX:ParallelGCThreads来设置。默认情况下为CPU的数量，对于8核以上的CPU默认值为CPU数量的5/8
    • GC Worker Start:GC线程开始工作的时间-距离JVM启动的时间(毫秒值),理想情况下所有的线程应该同一时间开始工作，Diff应该为0
    • Ext Root Scanning:扫描根节点所花费的时间，找到任何能够进入当前收集集的对象。
    • Update RS (Remembered/SetRSet)： 更新RS
    • Processed Buffers：在更新RS的过程中有多少 UpdateBuffers被处理了。
    • Scan RS (ms)：扫描每个区域的 Remembered Set找出在CS中记录的Region花费的时间。
    • Code Root Scanning (ms)：扫描跟对象，查找出在CS中引用的对像所花费的时间
    • Object Copy (ms)：对象疏散清理阶段，所有在CS中的Region都需要被清理。复制对象就是将哪些依然存活的对象复制到新的Region中去。
    • Termination：垃圾回收周期的结束阶段，重置RS，释放空闲区域，触发并发标记等耗时，一般情况下这个阶段的耗时时很少的。
    • GC Worker Other (ms)：表示花在未在先前任务中记录的其他任务上的时间。
    • GC Worker Total (ms)：显示每个并行工作线程所用时间的最小、最大、平均、差异和总和
    • GC Worker End (ms)：JVM 启动以来 GC 工作线程结束的最小/最大时间戳。差异表示第一个和最后一个线程结束之间的毫秒差。理想情况下，它们应该同时结束。
  • ③Code Root Fixup:遍历指向收集集的已标记方法，修复在 GC 过程中可能已移动的指针。
  • ④Code Root Purge:清除代码根目录中的条目
  • ⑤Clear CT: 清理脏RS中Card所花费的时间
  • ⑥ 其他串行任务
    • Choose CSet：选择哪些Region可以加入CSet所花费的时间
    • Ref Proc：处理 STW 引用处理器发现的软/弱/终结/虚/JNI 引用花费的时间。
    • Ref Enq：循环引用并将它们加入待处理列表。
    • Redirty Cards：在垃圾回收过程中当一个对象被移动到另一个Region或者其引用关系发生变化时，相应的Card就会被标记为脏。
    • Humongous Register：启用 G1ReclaimDeadHumongousObjectsAtYoungGC（默认启用/JDK 8u60 中新增功能）时，G1 会尝试在年轻代 GC 期间收集巨大区域。该操作表示评估巨大区域是否符合回收条件并记录它们所需的时间
    • Humongous Reclaim： 确保巨大对象死亡并清理的时间，释放区域，重置区域类型，将区域返回到空闲列表并记入已释放空间。
    • Free CSet：被清空的区域被重新加入空闲列表
  • ⑦
    • Eden：显示当前年轻代收集的触发原因是 Eden 空间已满，分配了1214.0M。所有的Eden区都清理完毕使用量为0M。
    • Survivors: 14.0M->16.0M：由于年轻代转移，存活区从 13.0M 增加到 139.0M。
    • Heap: 1531.6M(2048.0M)->318.7M(2048.0M)：收集前，总堆分配为 1531.6M（最大 2048.0M）。收集后总堆分配减少至318.7M 最大堆保持不变。
  • ⑧ 收集所用时间
    • user=0.02：在收集中花费在用户代码的 CPU 时间，包括所有CPU上的所有线程。未计入进程外的时间或等待时间。根据并行线程数量，用户时间会远高于实际时间。
    • sys=0.00：在进程内核中花费的 CPU 时间
    • real=0.00：从收集开始到结束的真实时钟时间。这还包括在其他进程中花费的时间和等待时间。
